## Введение в техническое документоведение.
## Виды технической документации для разработчиков и тестировщиков 
______________________________________________________________________
**Документация** на программное обеспечение - печатное руководство пользователя, диалоговая документация и справочный текст, описывающий как пользоваться ПО.

*документ* - целевая информация, предназначенная для конкретной аудитории, размещенная на конкретном носителе в заданном формате.

_Программная документация_ - документы, содержащие в зависимости от назначения данные, необходимые для разработки, производства, эксплуатации и сопровождения программы.
________________________________________
## Основные типы документации ПО
1. Архитектурная( проектная ) - обзор ПО, включающий описание рабочей среды и принципов, которые должны быть исполнены при создании ПО.
2. Техническая документация - документы на код, алгоритмы, интерфейсы, API и структуры данных.
3. Пользовательская  документация - руководство для дальнейших пользователей, администраторов системы и прочего персонала. Существует три подхода организации пользовательской документации:
	1. Пользовательское руководство(tutorial) - пошаговое описание действий, необходимых для выполнения каких-либо личных задач. Подходит для начинающих пользователей 
	2. Тематический подход - заключается в разделении тем на главы, подходит для совершенствующихся пользователей.
	3. Алфавитный справочник - заключается в организации задач или команд в алфавитном порядке. Подходит для продвинутых пользователей, которые знают заранее, что же им нужно найти.
4. Маркетинговая документация - цель данной документации в том, чтобы подогреть интерес к продукту у потенциальных пользователей, проинформировать их о функционале продукта, и объяснить положение продукта по сравнению с конкурентами.
