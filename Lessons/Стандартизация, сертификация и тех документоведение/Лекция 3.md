# Основа сертификации программного обеспечения Процедуры сертификации для программистов и тестировщиков 

сертификация ПО - документальное подтверждение соответствия программного продукта определенным критериям качества, безопасности, надежности и совместимости.
Она может быть обязательной или добровольной, в зависимости от типа продукта, назначения и требований рынка. 

Сертификация включает всестороннюю проверку кода, функциональности, совместимости, защиты данных, производительности и других важных характеристик. 

## Основные типы сертификации ПО включают: 
- функциональная сертификация проверка на соответствие функциональным требованиям 
- Сертификация безопасности - оценка на предмет защиты от потенциальных угроз и уязвимостей 
- Сертификация совместимости -тестирование работы ПО в различных средах.
- Сертификация производительности - анализ быстродействия и эффективности использования ресурсов. 

# Зачем нужна сертификация
 - для повышения доверия пользователей - дает пользователям уверенность что ПО соответствует определенным стандартам и повышает  уровень доверия к продукту
 - Соответствие законодательным требованиям - в ряде отраслей, как медицина, транспорт и финансы, сертификация обязательна
 - Снижение рисков - продукты прошедшие сертификацию чаще демонстрируют стабильность в работе, что снижает риски возникновения технических ошибок 
 - Выход на международный рынок
# Этапы Сертификации: 
1. подготовительный - компании проводят внутренний аудит, чтобы оценить готовность ПО к сертификации 
2. Анализ и планирование - на этом этапе разрабатывается план сертификации, в котором указываются все необходимые тесты, проверки и документации.
3. Тестирование и проверка - специалисты проводят тестирование ПО на соответствие стандартам качества, безопасности, надежности и совместимости. 
4. Получение сертификата - после успешного завершения всех проверок компания получает сертификат, подтверждающий соответствие продукта необходимым требованиям.
### Качество ПО 
это степень, в котором ПО соответствует потребностям или ожиданиям его пользователей.

# Критерии и качество ПО
1. Критерий надежности - устойчивость в функционировании, заключается в способности ПО продолжить свою работу после возникновения отклонения, которые могут вызваны техническими сбоями, ошибками входных данных или обслуживанием.
2. Критерий сопровождения - сопровождение ПО определяется четырьмя критериями:
	1. структурность.
	2. простота конструкции.
	3. наглядность.
	4. повторяемость .
3. Критерий удобства применения ПО - характеризуется через легкость в его освоении. 
4. Критерий эффективности - главные критерии эффективности считают:
	1. Уровень автоматизации функций обработки данных.
	2. Временная эффективность. 
	3. Ресурсоемкость 
5. Критерий универсальности - заключаются в: 
	1. Гибкости 
	2. Мобильности
	3. модифицированности 
# Методы контроля качества ПО:
- Методы и техники, связанные с выяснением свойств ПО во время его работы.
- Методы и техники определения показателей качества на основе симуляции работы ПО с помощью моделей разного рода.
- Методы и техники, нацеленные на выявления нарушений формализованных правил построения исходного кода проектных моделей и документации.
- Методы и техники обычного или формализованного анализ проектной документации и исходного кода, для выявления их свойств.
