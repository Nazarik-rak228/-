## . Введение в ООП

- **Определение**: ООП — подход к программированию, где программа строится как набор объектов. Каждый объект — экземпляр класса. Классы образуют иерархию наследования.
- **Класс**: Шаблон ("чертеж") для объектов. Определяет атрибуты (данные) и методы (операции).
- **Объект**: Сущность с атрибутами (свойствами) и методами (поведением). Экземпляр класса.
    - Пример: Класс "Автомобиль" — общая схема. Объект — конкретный "Лада".
- **Аналогия**: Класс = чертеж автомобиля. Объект = реальный автомобиль на дороге.
- **Свойства объекта**: Атрибуты (состояние, "какой объект") — данные.
- **Поведение объекта**: Методы (функции, привязанные к объекту, "что объект может делать").

## 2. Синтаксис создания класса в Python

- **Базовый синтаксис**:
    
    Python
    
    ```
    class ИмяКласса:  # CamelCase по соглашению
        pass  # Если пустой класс
    ```
    
- **Магический метод **init**** (конструктор):
    - Вызывается автоматически при создании объекта.
    - Инициализирует атрибуты.
    - Принимает параметры для начальных значений.
    - Пример: Установка начальных данных объекта.
- **Параметр self**:
    - Ссылка на экземпляр объекта.
    - Всегда первый параметр в методах.
    - Python автоматически передает объект как self (например, obj.method() = Класс.method(obj)).
    - Не ключевое слово, но по соглашению — всегда self (не меняй!).

## 3. Атрибуты объектов

- **Определение**: Переменные, хранящие данные для каждого экземпляра.
- **Инициализация**: В **init**.
- **По умолчанию**: Могут иметь значения без передачи параметров.
- **Изменение**: Через прямой доступ или методы.
- **Антипаттерн**: Не добавляй атрибуты динамически вне **init**.
- **Типы атрибутов**:
    
| Тип                            | Описание                                                                               | Пример            |
| ------------------------------ | -------------------------------------------------------------------------------------- | ----------------- |
| Атрибуты объектов (экземпляра) | Уникальны для каждого объекта. Определяются в **init** через self.                     | self.name = "Имя" |
| Атрибуты класса                | Общие для всех объектов. Определяются вне методов. Изменение влияет на все экземпляры. | class_var = 10    |
- **Сравнение атрибутов**:
    
| Аспект       | Атрибуты класса                                | Атрибуты объектов                        |
| ------------ | ---------------------------------------------- | ---------------------------------------- |
| Определение  | Вне методов                                    | В **init** через self                    |
| Доступ       | Через класс или объект                         | Только через объект                      |
| Уникальность | Общие для всех                                 | Уникальные для каждого                   |
| Применение   | Константы, счетчики (e.g., кол-во экземпляров) | Персональные данные (e.g., имя, возраст) |
    
- **Когда использовать**:
    - Объекта: Для уникальных данных (имя, возраст, ID).
    - Класса: Для общих (константы, настройки по умолчанию).

## 4. Методы объектов

- **Определение**: Функции внутри класса, описывают поведение.
- **Доступ**: Через self к атрибутам.
- **Типы**:
    - Изменяющие состояние: Меняют атрибуты, возвращают None.
    - Возвращающие информацию: Не меняют, возвращают данные.
- **Совет**: Методы короткие, на одну задачу.
- **Типы методов в Python**:

| Тип                      | Описание                                  | Декоратор     | Первый параметр | Применение                                    |
| ------------------------ | ----------------------------------------- | ------------- | --------------- | --------------------------------------------- |
| Обычные (экземпляра)     | Работают с объектом.                      | Нет           | self            | Изменение/чтение атрибутов объекта.           |
| Классовые                | Работают с классом.                       | @classmethod  | cls             | Альтернативные конструкторы, атрибуты класса. |
| Статические              | Утилитарные функции, связанные с классом. | @staticmethod | Нет             | Вспомогательные, без доступа к self/cls.      |
| Специальные (магические) | Специальное поведение (e.g., **init**).   | Нет           | self            | Перегрузка операторов, стандартные операции.  |

- **Разница обычного и статического**: Обычный имеет доступ к self (объекту), статический — нет, как простая функция в классе.
- **Перегрузка операторов**: Определение, как операторы (+, -, ==) работают с объектами класса через специальные методы.
    - Без перегрузки: Нужно вызывать методы явно (e.g., obj.add(other)).
    - С перегрузкой: obj + other.
    - Нет перегрузки методов как в Java (имитация через args).
- **Часто используемые специальные методы**:

| Метод    | Описание                            | Пример        |
| -------- | ----------------------------------- | ------------- |
| **init** | Конструктор                         | obj = Class() |
| **str**  | Строковое представление (для print) | print(obj)    |
| **repr** | Для разработчика                    | repr(obj)     |
| **len**  | Длина                               | len(obj)      |
| **add**  | +                                   | obj + other   |
| **eq**   | ==                                  | obj == other  |

## 6. Основные принципы ООП

- **Четыре столпа**: Наследование, Абстракция, Инкапсуляция, Полиморфизм.
- **Аналогия**: Класс "Кошка" с атрибутами (окрас, порода, возраст) и методами (спать). Экземпляры — разные породы.

### 6.1. Наследование

- **Определение**: Дочерние классы перенимают атрибуты/методы от родительских, добавляя свои.
- **Пример**: "Домашняя кошка" наследует от "Кошка", добавляет "хозяин", "кличка", метод "клянчить вкусняшку".
- **Типы**:

| Тип            | Описание                        | Пример                                 |
| -------------- | ------------------------------- | -------------------------------------- |
| Одиночное      | От одного родителя.             | Cat от Animal.                         |
| Многоуровневое | Цепочка (Дед → Папа → Ребенок). | Smartphone от MobilePhone от Phone.    |
| Иерархическое  | Один родитель, много детей.     | Dog и Cat от Animal.                   |
| Множественное  | От нескольких родителей.        | Amphibian от WaterAnimal и LandAnimal. |

    
- **super()**: Вызов метода родителя для расширения (e.g., super().**init**()).
- **Проверки**: isinstance(obj, Class) — объект экземпляр класса? issubclass(Child, Parent) — подкласс?
- **Абстрактные методы**: В абстрактных классах (из abc) — шаблоны, реализуются в подклассах.

### 6.2. Инкапсуляция

- **Определение**: Скрытие деталей реализации, защита данных, контролируемый интерфейс.
- **Аналогия**: Автомобиль — водитель видит руль/педали, не механизм.
- **Модификаторы доступа** (соглашения, не строгие):

| Модификатор | Префикс | Доступ                            | Аналогия                    |
| ----------- | ------- | --------------------------------- | --------------------------- |
| Public      | Нет     | Отвсюду                           | Дверь дома.                 |
| Protected   | _       | В классе/подклассах (соглашение). | Комнаты в доме (для семьи). |
| Private     | __      | Только в классе (name mangling).  | Сейф в банке.               |

- **Геттеры/Сеттеры**: Методы для чтения/записи атрибутов с проверками.
- **Properties**: Декоратор @property — делает метод как атрибут (e.g., obj.value вместо obj.get_value()).
    - @value.setter для установки.
- **Задание: Класс BankAccount** — С балансом, инкапсуляцией (private баланс, геттер/сеттер с проверками).

### 6.3. Полиморфизм

- **Определение**: Объекты разных классов отвечают на одинаковые вызовы по-разному (единый интерфейс).
- **Типы**:

| Тип                            | Описание                                     | Пример                                 |
| ------------------------------ | -------------------------------------------- | -------------------------------------- |
| Подтипов                       | Через наследование, переопределение методов. | Animal.sound() — Dog "гав", Cat "мяу". |
| Утиная типизация (Duck Typing) | Не тип, а наличие методов.                   | Если есть swim() и quack() — работает. |
| Параметрический                | С дженериками (typing.Generic).              | Функции для разных типов.              |
| Ad-hoc                         | Перегрузка операторов.                       | + для чисел/строк.                     |

- **С функциями**: len() работает с list/str/dict.
- **Через наследование**: Базовый класс с методом, переопределение в детях.
- **Абстрактные классы**: Для строгого интерфейса.
- **Преимущества**: Расширяемость, меньше if, лучше тестирование, модульность.
- **Задание: Полиморфизм**

    2. Система оплаты: Способы с разной комиссией, но метод pay().
