**Конспект лекции №3: Методы, перегрузка и рекурсия в Java**

**1. Методы**

- **Определение**: Именованный блок кода, объявляемый внутри класса. Может использоваться многократно.
- **Структура**:

java

public static int maxFinder(int a, int b) {

    int max;

    if (a < b) max = b;

    else max = a;

    return max;

}

- **Вызов метода**:

java

int summary = maxFinder(number, 5);

---

**2. Типы методов**

**2.1. По возвращаемому значению**

- **Возвращаемое значение**:

- Указывается тип перед именем метода.
- Используется return для возврата значения.
- Пример:

java

public static int sum(int x, int y) {

    return x + y;

}

- **Void-методы**:

- Не возвращают значения.
- Выполняют операции (например, вывод на экран).
- Пример:

java

public static void maxFinder(int a, int b) {

    int max = (a < b) ? b : a;

    System.out.println(max);

}

**2.2. По модификаторам доступа**

Убывание уровня закрытости:

1. **private** — доступ только внутри класса.
2. **default (package private)** — доступ внутри пакета.
3. **protected** — доступ внутри пакета и у наследников.
4. **public** — доступ отовсюду.

---

**3. Параметры методов**

- Методы могут принимать данные через параметры.
- Рекомендуется использовать не более 7 параметров.
- Пример:

java

public static void maxFinder(int a, int b) {

    // Тело метода

}

---

**4. Перегрузка методов (Overloading)**

- **Определение**: Использование методов с одним именем, но разными типами и/или количеством параметров.
- **Важно**: Тип возвращаемого значения **не влияет** на перегрузку.
- **Пример корректной перегрузки**:

java

static int sum(int x, int y) { return x + y; }

static double sum(double x, double y) { return x + y; }

static int sum(int x, int y, int z) { return x + y + z; }

- **Пример некорректной перегрузки**:

java

static int sum(int x, int y) { return x + y; }

static double sum(int x, int y) { return x + y; } // Ошибка компиляции!

---

**5. Рекурсивные алгоритмы**

- **Определение**: Функция, вызывающая саму себя.
- **Базис рекурсии**: Условие выхода из рекурсии (решение задачи без рекурсии).
- **Шаг рекурсии**: Вызов функцией самой себя с изменёнными параметрами.
- **Пример корректной рекурсии (факториал)**:

java

private static int factorial(int n) {

    if (n == 0 || n == 1) return 1; // Базис

    return n * factorial(n - 1);     // Шаг рекурсии

}

- **Пример неправильной рекурсии** (бесконечный цикл):

java

public void recursionFunc() {

    System.out.println("Привет!");

    recursionFunc(); // Бесконечный вызов!

}

---

**Итог**

- **Методы** упрощают структуру программы и позволяют повторно использовать код.
- **Перегрузка** улучшает читаемость, позволяя использовать одно имя для разных операций.
- **Рекурсия** полезна для задач, которые можно разбить на подзадачи, но требует обязательного условия выхода (базиса).
