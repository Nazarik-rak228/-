   # **Техническое задание

## **1. Название проекта и краткое описание**

**Randomizer Desktop** — настольное приложение для генерации случайных чисел и случайного выбора элемента из пользовательского списка.

---

## **2. Цель и назначение**

Цель проекта — создать простую программу для компьютера, позволяющую пользователю быстро получать случайные числа или выбирать случайный элемент из списка.

Программа может использоваться для:

- учебных задач
    
- случайного выбора (например, участника)
    
- игр и розыгрышей.
    

---

## **3. Область применения / пользователи**

Приложение предназначено для обычных пользователей ПК:

- студенты и школьники
    
- преподаватели
    
- пользователи, которым нужен простой инструмент случайного выбора.
    

---

## **4. Функциональные требования**

### Обязательные

1. Пользователь может ввести **минимальное и максимальное число**.
    
2. Программа генерирует **случайное число в заданном диапазоне**.
    
3. Пользователь может **ввести список элементов** (несколько строк).
    
4. Программа может **случайно выбрать один элемент из списка**.
    
5. Результат отображается в окне программы после нажатия кнопки **«Сгенерировать»**.
    

### Желательные

6. Кнопка **«Сгенерировать снова»**.
    
7. Отображение **истории последних 5 результатов**.
    
8. Возможность **копировать результат в буфер обмена**.
    

---

## **5. Нефункциональные требования**

- **Производительность:** генерация результата должна занимать не более **1 секунды**.
    
- **Надёжность:** программа не должна завершаться с ошибкой при неправильном вводе.
    
- **Совместимость:** приложение должно работать в ОС
    
    - Microsoft Windows
        
- **Удобство использования:** простой и понятный интерфейс.
    
- **Локализация:** интерфейс на русском языке.
    

---

## **6. Входные / выходные данные**

### Входные данные

- минимальное число (целое число)
    
- максимальное число (целое число)
    
- список элементов (строки текста)
    

### Выходные данные

- случайное число (целое число)
    
- случайный элемент списка (строка)
    

---

## **7. Интерфейсы**

Тип интерфейса: **графический интерфейс настольного приложения (GUI)**.

Основные элементы интерфейса:

- поле ввода **минимального числа**
    
- поле ввода **максимального числа**
    
- текстовое поле для **ввода списка элементов**
    
- кнопка **«Сгенерировать»**
    
- поле **отображения результата**
    

---

## **8. Ограничения и допущения**

- приложение разрабатывается как **простая настольная программа**
    
- разработка выполняется **одним разработчиком**
    
- используется язык программирования Python
    
- срок разработки — **1 неделя**
    
- приложение не требует подключения к интернету.
    

---

## **9. Критерии приёмки**

Продукт считается принятым если:

1. генерация числа работает корректно для любых допустимых диапазонов
    
2. выбор элемента из списка работает без ошибок
    
3. интерфейс корректно отображается на экране
    
4. программа не завершает работу при неправильном вводе
    
5. успешно проходит не менее **90% тест-кейсов**
